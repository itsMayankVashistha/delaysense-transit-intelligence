## Folder Intent

### Root files

- `.env`: local runtime configuration for API key and database path
- `README.md`, `README_org.md`: project description and setup notes
- `requirements*.txt`, `environment.yml`: environment management
- `Makefile`: environment setup helper

### `.github/workflows/`

- project automation around issue and PR status movement

### `app/`

- the demo application layer
- contains FastAPI, Streamlit, runtime artifacts, and lightweight inference services

### `data/`

- tracked data-related folder
- currently contains a large parquet file and placeholder structure
- note: several runtime-generated files referenced by scripts are not committed here

### `docs/`

- architecture, planning, and integration documentation

### `images/`

- presentation or documentation visuals

### `modeling/`

- model training and prediction code
- currently appears partially legacy/template-based rather than fully aligned with the TfL app pipeline

### `models/`

- placeholder folder for trained models

### `notebooks/`

- exploration, EDA, and modeling experiments

### `scripts/`

- operational pipeline scripts for station fetch, data collection, backup, dataset build, and DB inspection

## Important Untracked/Generated Runtime Files Referenced By Code

These files are part of the repo flow even if they are not currently visible in the committed tree:

- `data/raw/stations.csv`
- `data/raw/tfl_arrivals.sqlite`
- `data/raw/tfl_arrivals_final_backup.sqlite`
- `data/processed/dataset_192H.parquet`
- `data/processed/baseline_table_192H.parquet`

These are generated by the scripts and are important to understanding the actual project workflow.
