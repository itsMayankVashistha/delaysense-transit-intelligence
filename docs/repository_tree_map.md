## Directory Intent

- `app/`: deployed product code for FastAPI, Streamlit, runtime services, and packaged model artifacts.
- `scripts/`: data ingestion and dataset-building utilities for the TfL pipeline.
- `data/`: processed datasets and local data artifacts used for feature generation and intelligence lookup.
- `modeling/`: mixed modeling area; `validate_model_artifact.py` is aligned with the app, while `train.py` and `predict.py` still look template or legacy.
- `models/`: older or external model artifacts, notebooks, and modeling contract files.
- `docs/`: project documentation, contracts, reports, and handoff material.
- `notebooks/`: EDA and experimentation notebooks.
- `images/`: presentation or documentation images.
- `catboost_info/`: local training logs generated by CatBoost.

## Important Notes

- `app/api/routes.py` currently exists but is empty; the live API is implemented in `app/api/main.py`.
- `app/tests/` exists but is empty right now.
- `app/artifacts/model.joblib` and `app/models/*.joblib` represent two model-storage conventions inside the repo. The active runtime path in `app/config/settings.py` points to `app/models/lightgbm_v2_h300_balanced.joblib`.
- Several files under `data/`, `catboost_info/`, and `models/catboost_info/` are generated artifacts rather than handwritten source code, but they are part of the present repository shape.
